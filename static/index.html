Objectif.AI User Guide

v0.7.8 — Open source under AGPL-3.0 · Source code

Quick Start

  1. Run start.bat (or setup-service.bat for auto-start on login)
  2. Open http://localhost:32168 — dashboard loads automatically
  3. Go to Model Browser, find a model, click Download Model
  4. Once downloaded, click Load Model
  5. In BlueIris: AI → Configure → URL: http://<this-machine-ip>:32168

Installation

Manual (start.bat)

Double-click start.bat. Server runs while the console window is open. Good for testing.

As a Scheduled Task (setup-service.bat)

Run setup-service.bat once. Objectif.AI starts automatically on Windows login with no console window. The Restart Server button in Settings becomes available when the task is installed. To remove it, run remove-service.bat or use Settings → Server → Remove from Startup.

The task runs as your Windows user account, giving it full GPU access. Restarts automatically on crash.

API Key & Security

On first run a random API key is generated and saved to config.yaml. The server embeds it into the dashboard page automatically — no entry required from any device on your network.

The key protects management endpoints. The BlueIris detection endpoint (/v1/vision/detection) does not require the key.

Keep config.yaml private — it contains your API key. Never commit it to a public repository.

BlueIris Configuration

In BlueIris → AI → Configure: Server = Custom, URL = http://<ip>:32168, Path = /v1/vision/detection.

Set Objectif.AI's minimum confidence lower than your BlueIris threshold so BlueIris receives all candidate detections and decides what to trigger on.

Downloading and Loading Models

Every model follows the same flow regardless of type:

  1. Download Model — downloads the weights file to the models/ folder. A progress bar appears during download.
  2. Load Model — loads the downloaded model into memory for inference.
  3. Active — model is serving detections to BlueIris.

YOLO v5 and Torchvision models (Faster R-CNN, SSD, etc.) show an indeterminate progress bar during download — PyTorch does not expose download progress for these. The download is still happening.

Small or Medium works best for most security camera setups. Start there and move up only if you are missing detections you care about.

The Console

Each incoming image gets a shape symbol (●, ■, ▲, ◆, ★…). The matching detection result uses the same shape — lets you trace a request to its result even when multiple images arrive at the same time.

  • IN — image received from BlueIris
  • OUT — detection result returned
  • SYS — server status messages
  • WARN — warnings
  • ERR — errors

Toggle between expanded (each item on its own line, color-coded by confidence) and compact display in Settings → Display.

Hardware & GPU Setup

NVIDIA CUDA

Click Install PyTorch CUDA on the Hardware tab (~1.8 GB). After it finishes, run python -m pip install onnxruntime-gpu, then restart.

Legacy GPU — CC below 7.5 (GTX 10-series, Tesla P4, GTX 1060)

Handled automatically. On first YOLO model load, the model is exported to ONNX (up to 60 seconds — progress shown in console) and cached. Subsequent loads are instant. Requires onnxruntime-gpu — install via the Dependencies tab.

YOLOv5 and Torchvision models cannot use this path and fall back to CPU.

Intel OpenVINO

Click Install OpenVINO on the Hardware tab (~200 MB). Works on Intel Arc and Intel integrated graphics.

AMD ROCm

Experimental, not officially tested. See rocm.docs.amd.com.

Dependencies Tab

Shows every package Objectif.AI uses, whether it is installed, and provides install buttons where applicable. Check here first if something is not working.

Detection Settings

Minimum Confidence: Server-side floor — detections below this are not sent to BlueIris. Default 30%. Set lower than your BlueIris trigger threshold.

Class Filter: When enabled, only selected COCO classes are returned. Click a category header to toggle all classes in that group at once.

Themes

10 themes in Settings → Display: Dark, Light, Solarized, Dracula, Nord, Gruvbox, Monokai, Ocean, Forest, Rose, Hi-Contrast. Saved in config.yaml, applies to all browsers.

Troubleshooting

BlueIris not getting detections

  • Check a model is loaded (green dot in header)
  • Confirm BlueIris URL matches this machine's IP on port 32168
  • Console — are IN/OUT lines appearing when BlueIris sends images?
  • Lower the minimum confidence threshold in Settings
  • Disable the class filter to rule it out

GPU not being used

  • Hardware tab — check status dots and package badges
  • Dependencies tab — check for missing packages
  • Use Set as Backend on the Hardware card
  • Restart server — model reloads on new backend automatically

Model shows Download Model after already downloading

For Torchvision models, PyTorch saves to a subfolder under models/ that varies by version. Refresh the Model Browser — if the file exists anywhere under models/ it should be detected automatically.

Legacy GPU first load is slow

ONNX export takes up to 60 seconds per model, once only. Watch the console for progress. Subsequent loads are instant.

YOLOv5 fails to load

Needs seaborn, pandas, matplotlib — usually installed automatically. If not: pip install seaborn pandas matplotlib

Scheduled task not starting

  • Check logs/startup.log
  • Open Windows Task Scheduler → find ObjectifAI → check Last Run Result
  • Run setup-service.bat again to re-register

Server crashes on startup

Check logs/startup.log and logs/server.log. Common causes: port 32168 in use, missing package (run pip install -r requirements.txt), corrupt model file (delete from models/ and re-download).

Updating

Extract the new zip, copy all .py files and static/index.html over your existing folder. Do not overwrite config.yaml or the models/ folder. Restart the server.
